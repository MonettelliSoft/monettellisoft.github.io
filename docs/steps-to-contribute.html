Steps to contribute

Do you like Xamarin Community Guide and want to help improve it? Brilliant! We are working to make this process as easy and transparent as possible. It is possible that we're not there yet, but this guide will help you grow as a contributor and will provide everything you need to make your first contribution. If any are missing information that prevents you from submitting a pull request, please let us know. We treat these types of issues as real mistakes.

important

Everyone who participates in this project, either as a team or a contributor, is obliged to follow the projects Contributor code of conduct. Every violation against it will be reviewed and investigated and will result in a response that is deemed necessary and appropriate to the circumstances. The project team is obligated to maintain confidentiality with regard to the reporter of an incident. Further details of specific enforcement policies may be posted separately.

How to contribute#

Xamarin Community Guide offers a variety of ways to contribute. If you struggle to find something suitable for yourself, join the discord channel and communicate with the team. Don't be shy, they are there to help you!

You can participate by:

  • Creating publications to the Community Blog section.
  • Improving documentation.
  • Placing a star in the repository.
  • Help out in the Discord Channel support channel.
  • Spread the good news about the project on twitter using the hashtag #XamarinCommunityGuide and Xamarin.
  • Contributing to the source code.
  • Be part of the Xamarin Community Guide Team.
  • Contributing ideas in GitHub Discussions.

Creating publications to the Community Blog section#

We consider this contribution the most important because it allows us to structure the content of the documentation. Xamarin Community Guide fosters the collaborative spirit within you.

important
  • You must have a GitHub account.
  • Install the code editor of your choice (VS Code, Vim, Webstorm, Atom, etc ...)
  • Install Node.js version >= 12.13.0 or above (which can be checked by running node -v). You can use nvm for managing multiple Node versions on a single machine installed
  • Install Yarn version >= 1.5 (which can be checked by running yarn --version). Yarn is a performant package manager for JavaScript and replaces the npm client. It is not strictly necessary but highly encouraged.

The following instructions will help you create a publication and send its PR for its respective review, remember to follow it step by step, be very patient.

  1. Click on the GitHub icon.

  2. In the repository, give it a star (to motivate the team), then fork to your GitHub account.

  3. Clone the repository.

    note

    To make changes and preserve git history, you must clone the repository on your local development machine.

    In the following command, change YOUR-USERNAME to your GitHub nickname.

    git clone https://github.com/YOUR-USERNAME/xamarincommunityguide.github.io.git

    The following command, directs you to the repository copy.

    cd xamarincommunityguide.github.io
    important

    Make sure you are in the XamarinCommunityGuide-Core branch using the git branch command.

    Clone the repository

  4. Install the dependencies.

    Xamarin Community Guide requires certain dependencies to work, don't worry! All these dependencies are handled by the Yarn package manager, so you only need to run one command

    yarn install
    caution

    Computers using ARM architecture (for example: Mac with M1) may have problems installing dependencies, this is because the sass-style preprocessor is not fully adapted for that type of architecture yet.

    Install the dependencies

  5. Run the project.

    Yarn provides a command to start a server and restart the site as soon as you make any changes to the project source code, you can access the site from your browser by visiting the URL http://localhost:3000/ (default).

    tip

    To make the changes, use your favorite code editor, preferably Visual Studio Code.

    yarn start

    Run the project

  6. In the blog folder, in the author.yml file, add your public data as a contributor.

    caution
    • For better navigation, the description of the slug has to be identical to the file of its publication.
    • Make sure to add your created nickname in the authors description.
    • In the tags, in addition to those that represent your publication, add if it is in Spanish or English.

  9. Main structure of the publication.

    Xamarin Community Guide made two posts:

    • The Post Template: It is a group of components that you can use to enrich your publication.

    • My first post: It is a concept of how your post should be.

  10. Submit your pull request.

  • GitKraken Git GUI
  • Download and install GitKraken Git GUI.
  • Integrate with GitHub.
  • Go to the project repository.
  • Right click, and select Open with GitKraken.
  • Click on Stage all changes.
  • In the Commit Message, the Summary structure should be as follows:
Publication: My title of the publication

Done, if you did all the steps correctly and the Xamarin Community Guide Team accepted your post, you will be on our honor page as a contributor. Thank you very much.
